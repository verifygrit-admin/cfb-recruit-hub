When to Trigger
  • After redeploying Apps Script (Code.gs changes)
  • After modifying scoring.js or gate logic
  • After updating the GrittyOS DB sheet
  • Before merging any PR that touches auth, scoring, or data pipelines
  • When users report zero results or broken submissions
  • On-demand for a full system health check
Severity Definitions
Critical
Data loss, SAID collision, silent formula failure, broken API. Platform cannot be trusted until resolved.
Warning
Degraded UX, missing data for subset of schools, edge cases affecting some users.
Info
Minor gap, improvement opportunity, or unconfirmed risk requiring manual verification.
Audit Responsibilities
1. Auth & Data Integrity
  • Duplicate SAIDs, orphaned records, missing timestamps
  • SAID generation (GRIT-[Year]-[NNNN]) unique and sequential
  • Recruits tab column mapping (cols 1–23 profile, 24–26 status)
  • Magic link flows: ?completeSAID=&token= expiry logic
  • saveRecruit vs updateRecruit routing correctness
2. API Viability
  • VITE_API_BASE not stale, Apps Script returns JSON not HTML
  • CORS not blocking GitHub Pages domain
  • Code.gs modified but redeploy status uncertain
3. Data Pipeline Health
  • Missing/null: coordinates, division/tier, COA, admission_rate, graduation_rate, ADLTV
  • q_link (592/661 known), coach_link (552/661 known) — flag new gaps
  • unitid join key intact
4. Formula Integrity — GRIT FIT
  • Gate 1 Athletic Tier → Gate 2 Recruit Reach → Gate 3 Academic Fit
  • No 40-yard dash: gate 1 graceful failure?
  • Test-optional schools exempt from SAT scoring?
  • Zero-match athlete surfaced clearly?
  • Null ADLTV/COA: score 0 or unhandled exception?
5. Multiuser Collision Risk
  • Simultaneous submissions → same SAID?
  • Lock service guarding SAID counter and tab writes?
  • Race condition in pendingToken generation?
6. Surface Stability
  • Broken map markers, null popup data
  • Dead q_link / coach_link entries
  • Filter states returning zero results with no explanation
  • Short list persistence: sign out → sign in → restored

Semver Bump Rules
MAJOR x.0.0
Breaking changes, schema rewrites, API contract changes. Never bumped without explicit confirmation.
MINOR x.y.0
New features, new modules, new endpoints — backwards-compatible additions.
PATCH x.y.z
Bug fixes, copy/style changes, config updates, docs. Default when in doubt.
