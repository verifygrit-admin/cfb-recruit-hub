   - Confirm Recruits tab column mapping (cols 1-23 profile, 24-26 status) is intact
   - Verify magic link flows: ?completeSAID=&token= parameter handling, expiry logic
   - Confirm email change flow: 6-digit code → verify modal → old email freed

2. API VIABILITY
   - Verify VITE_API_BASE is not stale
   - Confirm Apps Script Web App responds with JSON (not HTML error pages)
   - Check CORS headers not blocking GitHub Pages domain

   - Flag if Code.gs modified but redeploy status uncertain

3. DATA PIPELINE HEALTH
   - Confirm school data loads from GrittyOS Master DB without errors
   - Flag missing/null fields that cause GRIT FIT Formula to fail silently:
     coordinates, division/tier, COA, admission_rate, graduation_rate, ADLTV,
     q_link (592/661 known), coach_link (552/661 known)
   - Confirm unitid join key is intact

4. FORMULA INTEGRITY (GRIT FIT Formula — src/lib/scoring.js)
   Gate 1 — Athletic Tier: tier classification executes first
   Gate 2 — Recruit Reach: reach/target/safety banding correct
   Gate 3 — Academic Fit: GPA, SAT/ACT, selectivity matching executes last
   Edge cases to flag:
     - No 40-yard dash value: does gate 1 fail gracefully?
     - Test-optional schools: exempt from SAT scoring?
     - Zero-match athlete: surfaced clearly or silent empty result?
     - Null ADLTV or COA: scores 0 or throws unhandled exception?

5. MULTIUSER COLLISION RISK
   - Two simultaneous submissions → same SAID?
   - Recruits tab auto-creation with concurrent requests?
   - Lock service guarding SAID counter and tab writes?
   - Race condition in pendingToken generation allowing token reuse?

6. SURFACE STABILITY
   - Broken map markers (missing coordinates, null popup data)
   - Dead q_link / coach_link entries
   - Result cards with missing/null fields
   - Filter states returning zero results with no explanation
   - Short list persistence: sign out → sign in → short list restored
   - SwitchModal and AuthModal render correctly on edge-case submission paths
