SEMVER RULES (how the agent decides the bump)
---------------------------------------------
  MAJOR (x.0.0)  Breaking changes, schema rewrites, API contract changes
  MINOR (x.y.0)  New features, new modules, new endpoints — backwards-compatible
  PATCH (x.y.z)  Bug fixes, copy/style changes, config updates, docs
  When in doubt → patch. Never bumps major without your explicit confirmation.


SAFETY RULES
------------
- Never tags without your "yes"
- Never force-pushes or overwrites existing tags
- Warns if there are uncommitted changes before tagging
- Always tags HEAD — never a historical commit unless you specify
